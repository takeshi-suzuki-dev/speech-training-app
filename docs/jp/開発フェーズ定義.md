# 開発フェーズ定義

本プロジェクトは、以下の4フェーズで開発を進める。

- **Phase 0**: PoC（技術検証）
- **Phase 1**: MVP（最小プロダクト）
- **Phase 2**: Release v1.0（学習価値の強化）
- **Phase 3**: Release v2.0（自由回答対応）

---

## Phase 0: PoC（Proof of Concept）

### 目的

- コア機能が技術的に成立するか確認
- 外部API（Azure AI / ElevenLabs）との連携検証

### スコープ

#### Azure AI（発音評価）

- 定型文の表示（固定）
- 録音開始 / 終了
- 音声＋テキストをAzureへ送信
- 評価結果取得

表示内容：

- 全体スコア
- 各評価項目（fluencyなど）
- 単語スコア
- 音素スコア
- 音素ごとの判定第一候補
- Azure APIレスポンス全文（rawレスポンス）

#### ElevenLabs（音声生成）

- 「見本音声生成」ボタン押下時のみAPI呼び出し
- 見本音声取得
- 「音声再生」ボタンで再生
- ElevenLabs APIレスポンス表示（音声データ除く）

### やらないこと

- DB保存
- フィードバック生成
- UI最適化
- 学習機能

### 成功条件

- 音声送信成功
- Azureから評価取得
- 各粒度のスコア取得（全体・項目別・単語・音素）
- Azure rawレスポンス確認
- ElevenLabsで音声生成成功
- 音声再生成功
- ElevenLabs rawレスポンス確認

---

## Phase 1: MVP（Minimum Viable Product）

### 目的

PoCをもとに最小限使えるプロダクトにする。

### スコープ

- 採点結果のDB保存
- 定型文の複数管理（DB）
- 採点結果を見やすく整理した単一画面
- 簡易ログイン画面（必要に応じて）
- Azure `RecognitionStatus` に応じた結果表示切り替え

### ゴール

最低限の発音トレーニングアプリとして利用可能な状態。

---

## Phase 2: Release v1.0（学習価値の強化）

### 目的

学習体験を強化し、継続利用の価値を提供する。

### スコープ

- 採点履歴の取得
- 折れ線グラフ・棒グラフによる進捗可視化
- 採点結果に基づくアドバイス
- ユーザー定型文追加
- ElevenLabsで見本音声生成＋DB保存
- 録音中の音量検出と送信前チェック

### ゴール

継続学習ができる実用アプリ。

---

## Phase 3: Release v2.0（自由回答対応）

### 目的

定型文練習から自由発話へ拡張する。

### スコープ

- テーマに基づく自由回答（2分以内）
- 録音 → 採点 → 評価表示の拡張
- UI統合 or 別画面は着手前に決定

### ゴール

実践的なスピーキング練習が可能なアプリ。

---

## 全体の流れ

| フェーズ | フォーカス           |
| -------- | -------------------- |
| Phase 0  | できるか確認する     |
| Phase 1  | 使えるようにする     |
| Phase 2  | 続けられるようにする |
| Phase 3  | 実践レベルにする     |

---

## この設計の意図

- 外部依存（Azure / ElevenLabs）を最初に検証
- MVPで過剰実装を避けている
- 学習価値を段階的に追加している
- 自由回答（重い機能）を後ろに逃がしている
