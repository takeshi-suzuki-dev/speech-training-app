# 開発フェーズ定義

## Phase 0: PoC — 完了

### 目的

発音トレーニングアプリの中核技術が成立するかを確認する。

### 実装済み範囲

- Azure AI Speech 発音採点連携
- ElevenLabs TTS連携
- transcript表示
- Azure PronScoreを使ったoverall score表示
- sentence-level scores表示
- word-level breakdown表示
- phoneme-level breakdown表示
- 第1 / 第2 phoneme candidateとscore表示
- raw Azure JSON viewer
- 見本音声再生
- 基本的なAPIエラー表示

---

## Phase 1: MVP実装 — 完了

### 目的

PoCで確認した技術要素を、最小限使える発音トレーニングアプリにまとめる。

### 実装済み範囲

- 固定定型文カテゴリ
- カテゴリごとの固定定型文
- テンプレートベースの練習フロー
- `display_text` / `scoring_text` / `sample_audio_text` の分離
- Roger固定の見本音声
- Supabase Storageを使った最小TTSキャッシュ
- Azure AI Speechによる発音採点
- 有効な採点結果の `training_attempts` 保存
- 認識・採点が成立していない結果を履歴保存しない制御
- 定型文ごとの最新score表示
- 専用履歴画面
- 総合得点（Pron）グラフ
- 各日の最後5回平均
- 5日移動平均
- 20日移動平均
- overall / accuracy / fluency / completeness / prosody のscore breakdownグラフ
- Practice / History ナビゲーション
- README / docs整備

### 主な設計判断

- Phase 1では本格認証を入れない
- browser-local `client_id` で履歴を紐づける
- ユーザー録音音声は保存しない
- 保存対象は見本音声のみ
- 固定・共通の見本音声はpublic URLで再生する
- 自由入力TTS APIはPhase 1のユーザーフローには含めない

---

## Phase 2: 認証 + ユーザー指定定型文

### 目的

ログインしたユーザーが、自分専用の練習文を安全に登録して練習できるようにする。

ユーザー指定定型文には、経歴・面接回答・志望理由・ポートフォリオ説明など、秘匿性の高い内容が含まれる可能性がある。  
そのため、Phase 2 は認証前提で設計する。

### スコープ

- ログイン機能
- ユーザー管理
- 認証済み `user_id`
- ユーザー所有の練習文
- 必要に応じたユーザー所有カテゴリ
- ユーザー指定定型文の作成 / 編集 / 削除
- ユーザー指定定型文の見本音声生成
- ユーザー指定定型文での発音採点
- ユーザーごとの履歴管理
- ユーザー固有データのアクセス制御
- ユーザー固有見本音声の保護
- 必要に応じたprivate bucket / signed URL / backend経由配信

### 対象外

- 課金システム
- プラン別機能制御
- お題への自由回答
- Interview FAQ practice
- 本格的なマネタイズ運用

### ゴール

ログイン後に、自分の面接回答や経歴説明文を安全に登録して練習できる状態。

---

## Phase 3: 料金プラン + 課金 + ITエンジニア用単語練習

### 目的

料金プラン、課金システム、プラン別機能制御を導入し、継続可能な有料価値を作る。

Phase 3では、どの機能を Free / Basic / Plus / Pro に分けるかを定義し、契約状態に応じて利用できる機能を制御する。

### スコープ

- Free / Basic / Plus / Pro のプラン設計
- 課金システム構築
- subscription status管理
- プラン別機能制御
- プランごとの採点回数制限
- ユーザー指定定型文を上位プラン機能にする
- ユーザー指定定型文の見本音声生成を上位プラン機能にする
- ユーザー指定定型文の採点・履歴を上位プラン機能にする
- ITエンジニア用単語練習
- 固定定型文・単語練習を無料または低価格プランの入口機能として扱う
- 透明性のあるプラン境界と価格設計

### プラン方向性の例

下位プラン:

- 固定定型文
- ITエンジニア用単語練習
- 少ない採点回数
- 基本履歴

上位プラン:

- ユーザー指定定型文
- ユーザー指定定型文の見本音声生成
- ユーザー指定定型文での採点
- 採点回数増加
- 面接回答練習としての価値

### ゴール

プラン境界・課金・有料価値が明確になり、継続可能なサービス設計になっている状態。

---

## Phase 4: お題に自由回答 / Interview FAQ practice

### 目的

固定文・ユーザー指定定型文からさらに進め、質問やお題に対して自分の回答を練習できるようにする。

Phase 4では、単なる文の発音練習ではなく、面接質問への回答練習や自由回答練習に拡張する。

### スコープ

- Interview FAQ practice
- お題・質問一覧
- 質問音声
- ユーザーの回答テキスト登録
- 回答見本音声生成
- 回答を話して採点
- 回答ごとの履歴
- 将来的なfree speaking mode

### 例

1. ユーザーが面接質問を選ぶ
2. 自分の回答テキストを登録する
3. 質問音声を聞く
4. 回答の見本音声を生成・再生する
5. 自分で回答を録音する
6. 発音採点を受け、履歴で改善を見る

### ゴール

ユーザーが自分専用の面接回答を練習し、より自由なスピーキング練習へ進める状態。

---

## 全体の流れ

| フェーズ | フォーカス                                       |
| -------- | ------------------------------------------------ |
| Phase 0  | 技術的に成立するか確認する                       |
| Phase 1  | 固定定型文練習を使える形にする                   |
| Phase 2  | 認証とユーザー固有の練習文を導入する             |
| Phase 3  | 料金プラン・課金・ITエンジニア向け練習を導入する |
| Phase 4  | Interview FAQ / 自由回答練習へ拡張する           |

---

## この設計の意図

- 外部依存（Azure / ElevenLabs）を最初に検証する
- Phase 1ではコアの採点・履歴ループに集中する
- 秘匿性の高いユーザー指定定型文を扱う前に認証を導入する
- ユーザー指定定型文練習を高付加価値機能として扱う
- 認証済みユーザーフローができてからプラン・課金を導入する
- 固定文・ユーザー指定定型文の流れが安定してから自由回答に進む
