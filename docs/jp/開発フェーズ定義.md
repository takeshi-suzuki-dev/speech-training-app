# 開発フェーズ定義

## Phase 0: PoC — 完了

### 目的

発音トレーニングアプリの中核技術が成立するかを確認する。

### 実装済み範囲

- Azure AI Speech 発音採点連携
- ElevenLabs TTS連携
- transcript表示
- Azure PronScoreを使ったoverall score表示
- sentence-level scores表示
- word-level breakdown表示
- phoneme-level breakdown表示
- 第1 / 第2 phoneme candidateとscore表示
- raw Azure JSON viewer
- 見本音声再生
- 基本的なAPIエラー表示

---

## Phase 1: MVP実装 — 完了

### 目的

PoCで確認した技術要素を、最小限使える発音トレーニングアプリにまとめる。

### 実装済み範囲

- 固定定型文カテゴリ
- カテゴリごとの固定定型文
- テンプレートベースの練習フロー
- `display_text` / `scoring_text` / `sample_audio_text` の分離
- Roger固定の見本音声
- Supabase Storageを使った最小TTSキャッシュ
- Azure AI Speechによる発音採点
- 有効な採点結果の `training_attempts` 保存
- 認識・採点が成立していない結果を履歴保存しない制御
- 定型文ごとの最新score表示
- 専用履歴画面
- 総合得点（Pron）グラフ
- 各日の最後5回平均
- 5日移動平均
- 20日移動平均
- overall / accuracy / fluency / completeness / prosody のscore breakdownグラフ
- Practice / History ナビゲーション
- README / docs整備

### 主な設計判断

- Phase 1では本格認証を入れない
- browser-local `client_id` で履歴を紐づける
- ユーザー録音音声は保存しない
- 保存対象は見本音声のみ
- 固定・共通の見本音声はpublic URLで再生する
- 自由入力TTS APIはPhase 1のユーザーフローには含めない

---

## Phase 2: ユーザー定型文 / 学習体験強化

### 目的

ユーザー自身の練習文を扱えるようにし、学習体験を強化する。

### スコープ

- ユーザー定型文追加
- 必要に応じたユーザー定型文カテゴリ
- ユーザー定型文向け見本音声生成
- 見本音声管理の拡張
- sentence/category/date range による履歴フィルタ
- より詳細な履歴分析
- 弱点分析
- 採点結果に基づくアドバイス
- 必要に応じた認証設計・実装
- ユーザー固有音声が必要になった場合の保護配信

---

## Phase 3: Free Speaking Support

### 目的

固定定型文を超えた、より自由なスピーキング練習に対応する。

### スコープ

- Free speaking mode
- Interview-answer practice mode
- ユーザー設定の回答テキスト
- 質問音声と回答見本音声
- より柔軟な採点・復習フロー
- マネタイズ導入時の利用回数制限

---

## 現時点では対象外

- 本格的な課金システム
- 大規模ユーザー管理
- 多言語展開
- 日本語学習者向け対応
